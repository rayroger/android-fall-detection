# 🛡️ Android Fall Detection — AI-Powered

An end-to-end fall detection system using pre-trained ML models from public datasets, deployed as a real-time Android app.

---

## 📁 Project Structure

```
fall-detection-project/
├── ml_training/
│   ├── train_model.py        # Data loading, feature extraction, model training
│   └── requirements.txt      # Python dependencies
│
└── android_app/
    └── app/src/main/
        ├── java/com/falldetection/
        │   ├── MainActivity.java          # UI: toggle monitoring, set contact
        │   ├── FallDetectionService.java  # Foreground service: sensors → TFLite → alert
        │   ├── FeatureExtractor.java      # Mirrors Python feature extraction exactly
        │   └── ScalerParams.java          # Loads scaler_params.json, applies z-score
        ├── assets/
        │   ├── fall_detection.tflite      # ← generated by train_model.py
        │   └── scaler_params.json         # ← generated by train_model.py
        ├── res/layout/activity_main.xml
        └── AndroidManifest.xml
```

## 🚀 Quick Start

### Step 1 — Download a Dataset

| Dataset  | URL | Notes |
|----------|-----|-------|
| **SisFall** | https://www.ncbi.nlm.nih.gov/pmc/articles/PMC5298771/ | 15 fall types, includes elderly subjects — recommended |
| **MobiFall** | https://bmi.hmu.gr/the-mobifall-and-mobiact-datasets-2/ | Smartphone-native (Samsung Galaxy S3) |
| **FallAllD** | https://ieee-dataport.org/open-access/fallalld | 26,420 files, multi-sensor |

### Step 2 — Train the Model

```bash
cd ml_training
pip install -r requirements.txt

# Using SisFall:
python train_model.py --dataset sisfall \
    --data_dir /path/to/sisfall \
    --epochs 30 \
    --output ../android_app/app/src/main/assets

# Using MobiFall:
python train_model.py --dataset mobifall \
    --data_dir /path/to/mobifall \
    --epochs 30 \
    --output ../android_app/app/src/main/assets
```

This produces two files in `android_app/app/src/main/assets/`:
- `fall_detection.tflite` — quantised model (~50–200 KB)
- `scaler_params.json` — StandardScaler mean/std for runtime normalisation

### Step 3 — Build & Run the Android App

1. Open `android_app/` in **Android Studio**
2. Confirm `assets/` contains both generated files
3. Build & run on a physical device (API 26+)
4. Enter an emergency contact number
5. Toggle **Enable Fall Detection**

## 🧠 How It Works

```
Accelerometer + Gyroscope (50 Hz)
          │
          ▼
  Sliding Window (100 samples = 2 sec, 50% overlap)
          │
          ▼
  FeatureExtractor.java
  (46 statistical + SMV features — mirrors Python exactly)
          │
          ▼
  ScalerParams.java  (z-score normalisation)
          │
          ▼
  TFLite Model  (Dense NN, quantised INT8)
          │
     confidence ≥ 0.75?
          │ YES
          ▼
  Wait 3 seconds — still no movement?
          │ YES (variance < 0.5 m/s²)
          ▼
  📱 SMS alert + Push notification
```

## ⚙️ Tuning Parameters

Edit these constants in `FallDetectionService.java` and `train_model.py` (keep in sync):

| Parameter | Default | Effect |
|-----------|---------|--------|
| `WINDOW_SIZE` | 100 samples | Larger = more context, more latency |
| `STEP_SIZE` | 50 samples | Smaller = more frequent checks, more CPU |
| `FALL_THRESHOLD` | 0.75 | Lower = more sensitive (more false positives) |
| `CONFIRM_DELAY_MS` | 3000 ms | Longer = fewer false alarms |
| `STILLNESS_THRESHOLD` | 0.5 m/s² | Lower = stricter stillness required |
